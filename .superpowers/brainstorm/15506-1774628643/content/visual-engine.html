EACH MAPPING HAS
source -- what drives it (fft band, envelope, mouse, note velocity)
target -- which Hydra param it controls
range -- min/max output values
smooth -- interpolation factor (0=instant, 1=glacial)
curve -- linear, exponential, or step

Design Decisions

  • Hydra runs as embedded engine via hydra-synth npm package -- not an iframe, direct API access
  • Custom shaders registered via setFunction() at startup -- sacredGeometry, tribalMask, glitchScan, particleField, voidPulse, ritualFire
  • 4 output buffers (o0-o3) available -- user can build layered compositions with blend/feedback between buffers
  • Mapping system is the bridge -- any audio signal (fft band, envelope, note velocity, mouse position) can drive any Hydra parameter through a configurable range/smooth/curve
  • Users create/delete/edit mappings freely in the Mappings panel section -- this is where "endless possibility" lives
  • Presets save the entire state: synth type, effects, visual chain, all mappings. Exportable as JSON
